# 文档导航与可理解性优化设计（2026-02-27）

## 目标

- 提供清晰的文档入口与阅读路径，降低“该看哪一份文档”的成本。
- 让新手、运维、开发三类读者在 60 秒内找到最相关的内容。
- 不改动现有技术内容，只优化导航与结构提示，避免信息漂移。

## 受众与路径

- 新手：从 README 进入，随后看配置与部署。
- 运维/发布：Runbook、安全与事故处置。
- 开发/改代码：架构边界、前端路由与模块边界、API 约束。

## 方案概述

- 新增 `docs/README.md` 作为文档总览索引：
  - 快速入口
  - 按角色阅读路径
  - 关键边界 TL;DR
  - 文档约定（guides / architecture / plans）
- 在主 README 的“文档导航”新增入口指向 `docs/README.md`。

## 不做的事

- 不重写已有指南内容。
- 不改变文档层级或移动文件。
- 不新增额外的运维/架构规范条款。

## 验收标准

- `docs/README.md` 作为单一入口，链接现有关键文档。
- README 中可明确看到“文档总览入口”。
- 所有链接指向现有文档，无新增破坏性路径。

## 风险与缓解

- 风险：文档入口与实际内容更新不同步。
- 缓解：索引只做导航，不重复技术细节；保持链接最小集。
