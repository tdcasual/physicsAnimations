# Ops Observability Thresholds

适用范围：发布后巡检、线上值班、性能/稳定性告警处置。

## 1. 数据源

统一使用：

- `GET /api/metrics`

关键字段：

- `http.latencyMs.p95`
- `http.statusCounts.5xx`
- `http.requestsTotal`

## 2. 阈值

### 2.1 延迟阈值（p95）

- 黄色阈值：`p95 > 180ms` 持续 5 分钟
- 红色阈值：`p95 > 300ms` 持续 3 分钟

### 2.2 错误阈值（5xx）

- 黄色阈值：5 分钟窗口内 `5xx` 占比 `>= 1%`
- 红色阈值：5 分钟窗口内 `5xx` 占比 `>= 3%` 或连续增长 3 个采样周期

> 5xx 占比计算：`(statusCounts.5xx 增量 / requestsTotal 增量) * 100%`

## 3. 触发动作

### 3.1 黄色阈值触发

1. 保留当前版本，不立即回滚。
2. 采集最近 10 分钟 `/api/metrics` 快照。
3. 查看容器日志与错误栈，定位是否集中在单路由。
4. 若 15 分钟内未恢复，升级为红色处置流程。

### 3.2 红色阈值触发

1. 立刻执行流量缓解：
   - 暂停高频后台写操作（如批量写路径）
   - 降低并发入口（反向代理限流）
2. 若 10 分钟内指标无回落，执行回滚：
   - 按 `ops-release-runbook.md` 回滚步骤切回上个稳定镜像。
3. 回滚后再次检查：
   - `p95` 回到黄色阈值以下
   - 5xx 占比回到 `1%` 以下

## 4. 发布后最小检查清单

1. `curl -sf http://127.0.0.1:4173/api/metrics | jq '.http'`
2. 记录当前 `p95` 和 `5xx` 初始值
3. 观察 5-10 分钟增量趋势
4. 若触发阈值，按本文件执行处置并记录时间线
